# 需求变更与扩展登记

## REQ-001：多环境配置区分

- 状态：已登记，计划在 Step 15 实现。
- 优先级：高。
- 目标：支持 `dev`、`test`、`prod` 三种运行环境。
- 方案：通过 `APP_ENV` 选择 `config.{env}.yaml`，敏感值放在 `.env.{env}`。
- 约束：三种环境使用独立业务配置、数据库和密钥，环境间不得串用数据。

### 验收标准

- `APP_ENV=dev` 时加载开发配置并通过核心流程冒烟测试。
- `APP_ENV=test` 时使用独立测试数据库并通过全部自动化测试。
- `APP_ENV=prod` 时加载生产配置，且真实密钥不进入 Git。
- 未设置或设置非法环境时给出明确错误，不静默加载错误配置。
